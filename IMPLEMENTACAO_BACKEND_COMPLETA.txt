🚀 INICIANDO A APLICAÇÃO
================================================================================

1. CLONAR REPOSITÓRIO:
   git clone https://github.com/KaioQuaresmadS/CasamentoBack
   cd template-backend

2. RESTAURAR PACOTES:
   dotnet restore

3. ATUALIZAR SECRETS (appsettings.Development.json):
   {
	 "Jwt": {
	   "Secret": "sua-chave-secreta-super-longa-com-minimo-32-caracteres-aqui"
	 },
	 "MercadoPago": {
	   "AccessToken": "seu-access-token-sandbox-aqui",
	   "WebhookSecret": "seu-webhook-secret-aqui"
	 }
   }

4. CRIAR MIGRATIONS:
   dotnet ef migrations add InitialCreate --project src/CasamentoAnaKaio.Infrastructure
   dotnet ef database update --project src/CasamentoAnaKaio.Infrastructure

5. EXECUTAR APLICAÇÃO:
   dotnet run --project src/CasamentoAnaKaio.Api

6. ACESSAR:
   http://localhost:5000/swagger (documentação OpenAPI)
   http://localhost:5000 (aplicação)

📞 SUPORTE E TROUBLESHOOTING
================================================================================

ERRO: "JWT Secret must be at least 32 characters long"
SOLUÇÃO: Atualizar Jwt:Secret em appsettings com mínimo 32 caracteres

ERRO: "Database connection failed"
SOLUÇÃO: Verificar ConnectionStrings:DefaultConnection
		Verificar SQL Server instalado e executando
		Executar migrations: dotnet ef database update

ERRO: "Mercado Pago SDK not found"
SOLUÇÃO: dotnet restore
		 Verificar versão do mercadopago-sdk

ERRO: "Arquivo bloqueado em compilação"
SOLUÇÃO: taskkill /IM dotnet.exe /F
		 dotnet clean
		 dotnet build

ERRO: "CORS error no frontend"
SOLUÇÃO: Verificar appsettings "Cors:AllowedOrigins"
		 Certificar que frontend URL está configurada
